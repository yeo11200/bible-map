# 출애굽기 1~40장 지도 설계

## 목표

출애굽기의 모든 장을 지도 학습 흐름에 포함한다. 지리적 장소가 반복되는 장을 억지로 서로 다른 핀으로 만들지 않고, 실제 사건 구간을 핀으로 표시한 뒤 선택한 핀 안에서 관련 장의 사건과 본문을 읽는다.

## 지도 구간

1. 이집트: 1~12장
   - 억압과 모세의 탄생, 부르심, 애굽 귀환, 열 재앙, 유월절, 라암셋 출발
2. 광야 여정: 13~18장
   - 에담, 바다 앞 진, 홍해, 마라, 엘림, 신 광야, 르비딤, 이드로의 방문
3. 시내산: 19~40장
   - 언약, 십계명과 율법, 성막 설계, 금송아지, 성막 제작, 성막 봉헌

## 사용자 경험

- 지도에는 장소 또는 사건 구간별 핀을 표시한다.
- 핀을 선택하면 해당 지점의 핵심 성경 구절과 함께 관련 장 목록을 보여 준다.
- 장 목록의 항목을 선택하면 그 장의 요약과 전체 본문 리더의 해당 장으로 이동한다.
- 1~40장 모두 정확히 하나 이상의 지도 구간에 속한다.

## 데이터와 API

- `ExodusContextItem`에 `chapterRange`와 장별 사건 목록을 추가한다.
- 서버의 `/api/books/exodus`는 기존 전체 40장 KorRV 본문과 확장된 지도 항목을 반환한다.
- 지도 GeoJSON에는 이집트 내부 사건과 시내산 체류 지점을 포함한 핀을 추가한다.

## 위치 표현 원칙

- 성경 본문만으로 현대 좌표를 확정할 수 없는 지점은 `추정 위치`로 표시한다.
- 이집트·광야·시내산처럼 넓은 구간에서 일어난 사건은 교육용 구간 핀으로 표현하며, 고고학적 확정 위치처럼 단정하지 않는다.
- 마라처럼 바다와 혼동될 수 있는 사건은 육상 추정점으로 표시한다.

## 검증

- API 테스트로 1장과 40장이 모두 지도 항목의 장 범위에 포함되는지 검증한다.
- 탐색기 테스트로 선택한 구간의 장 목록과 본문 리더 이동을 검증한다.
- 전체 테스트와 TypeScript 검사를 실행한다.
